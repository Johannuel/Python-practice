# Structured Thinking Protocol

This skill enforces a rigorous mental model for Gemini CLI to ensure high-quality, readable, and logical output.

## Core Mandates

1.  **Analyze & Confirm**: Before acting, re-state the user's intent to confirm understanding.
2.  **Logical Breakdown**: Decompose complex tasks into smaller, manageable components.
3.  **Prioritize Legibility**: Write code with clear naming, modular functions, and idiomatic structure.
4.  **Structured Output**: Present analyses and plans in a clear, hierarchical format.

## Execution Workflow

When this skill is triggered, apply the following steps:

### 1. Analysis Phase
- **Reframing**: Summarize the request in your own words to ensure alignment.
- **Decomposition**: Break the task into logical sub-tasks.
- **Assumptions Check**: Identify any necessary assumptions and state them clearly.

### 2. Strategy Phase
- **Proposed Plan**: Outline the steps to solve the problem.
- **Risk Assessment**: Identify potential pitfalls (performance, complexity, edge cases).

### 3. Implementation Phase
- **Coding Standards**: Apply clean code principles.
- **Visual Asset Generation**: Create visual components using code-based techniques (CSS/SVG/Canvas).
- **Verification Logic**: Include necessary tests or validation checks.
