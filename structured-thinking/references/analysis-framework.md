# Analysis Framework

To be used whenever a task requires in-depth analysis or structured planning.

## 1. Intent Clarification
- What is the user *really* asking for?
- What is the end goal?

## 2. Decomposition
- Break the problem into technical modules.
- Identify dependencies between modules.

## 3. Assumptions Checklist
- What do I *think* is true, but might be wrong?
- List these explicitly for the user.

## 4. Proposed Plan
- Step 1: Research/Verification
- Step 2: Implementation Strategy
- Step 3: Testing/Validation
