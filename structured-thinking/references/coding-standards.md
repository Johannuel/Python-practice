# Coding Standards

Guidelines for ensuring high-quality, maintainable, and readable code.

## 1. Structure
- Use modular design.
- Keep functions small and focused on a single task (Single Responsibility Principle).

## 2. Naming
- Variables and functions must be descriptive (e.g., `calculateTotalRevenue` instead of `calc`).
- Avoid cryptic abbreviations.

## 3. Readability
- Use comments to explain the *why*, not the *what*.
- Follow language-idiomatic structures (PEP 8 for Python).

## 4. Quality
- Every feature or bug fix must be accompanied by a validation mechanism (unit test, reproduction script).
